## Why Phase 2.5 is one sprint, not six

The kickoff anticipated:
- Sprint 2.5.1 stub swap + basic wire-up
- Sprint 2.5.2 camera permission flow
- Sprint 2.5.3 edit-before-save
- Sprint 2.5.4 snapshot save (Drafting 166 integration)
- Sprint 2.5.5 confidence telemetry
- Sprint 2.5.6 phase milestone

Reconnaissance found every one of those concerns already addressed in
production: scan-dish.tsx owns the camera + permission + AI decompose
+ store + add.tsx flow. The dev preview no longer needs to rebuild any
of it. Sprint 2.5.1 closes the phase by literally linking into the
working production path.

## Conditional hotfix queue (open until device verify)

Phase 2.5 closes assuming production scan-dish.tsx works as advertised.
If device verification (Sprint 2.3.6 hand-off) surfaces issues — e.g.,
the user expected the AI flow to inherit the search query as a hint,
or the v2 timeline doesn't pick up scan-dish meal log writes — a
follow-up hotfix sprint can address them. Issues to watch:

| Potential issue | Likely fix |
|---|---|
| Search query lost on `router.push` to scan-dish | Pass `?queryHint=…` and have scan-dish prefill its prompt context — minimal change to scan-dish.tsx |
| `add.tsx` writes via `useNutrition.addFood`; does `useMealLogTimeline` pick it up? | Should be automatic since add.tsx + useNutrition use the same nutrition repository; verify via device |
| `EmptyStateView` filter-clear secondary CTA still visible when AI CTA is the primary | Cosmetic only; tune copy if needed |
