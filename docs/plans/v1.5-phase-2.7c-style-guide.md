## 2. Naming convention (既 175 件 audit ベース)

### Strength exercises (V2 row、 `scripts/seed-exercises-v2/data.ts`)
- `name_ja`: カタカナ + 漢字 mix、 公式日本語名相当 (例: 「インクラインダンベルプレス」)
- 区切り文字: 連結 (中黒「・」 / ハイフン / 半角空白すべて使用しない)
- 器具明示が必要な場合: 接頭辞 (例: 「バーベルカール」 / 「ダンベルカール」 / 「マシン~」)
- バリエーション接頭辞: 「ワイド〜」「クローズ〜」「インクライン〜」「デクライン〜」「シングルアーム〜」 等
- `name_en`: タイトルケース、 半角空白区切り、 必要なら括弧で器具明示 (例: 「Bench Press (Barbell)」)
- `slug`: snake_case、 lowercase、 `<movement>_<equipment>` or `<variation>_<movement>_<equipment>` (例: `incline_bench_press_dumbbell`)

### Cardio / Sports / Other (Legacy row、 `src/constants/exercises.ts`)
- `nameJa`: 動作名 + 強度括弧 (例: 「ランニング（中、10km/h）」 「サイクリング（速め）」)
- `nameEn`: タイトルケース、 必要なら括弧で強度明示 (例: 「Running (moderate, 10 km/h)」)
- `metValue`: 2011 Compendium of Physical Activities 引用、 一般的な MET 値

## 6. Quality control 自己 check (agent generation 中の internal verify)

各 batch generation 後、 agent が下記 self-audit:
- (a) `name_ja` 重複 zero (既 175 + 新 batch unique)
- (b) `muscle_group` enum 値域内 (上記 §3)
- (c) `equipment` enum 値域内
- (d) `movement_pattern` enum 値域内 (strength のみ、 cardio/sports は不要)
- (e) `exerciseType` (cardio/sports/other) 値域内
- (f) Brand-name / trademark 不在
- (g) Naming convention (§2) 厳守
- (h) `primary_muscle` 値が §3 pattern と整合 (strength のみ)
- (i) `metValue` (cardio/sports) が realistic range (1.5-15)
- (j) `rep_range_low/high` (strength) が realistic range (1-50)
- (k) `is_compound` boolean (strength のみ、 compound vs isolation 判定正確)

self-audit fail なら該当 entries を修正 / 削除、 全 clear で ingestion へ。
