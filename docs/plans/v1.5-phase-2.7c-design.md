## Implementation 想定 (Sprint 2.7c.2-3 で literal)

### Sprint 2.7c.1 recon で確認すべき current codebase reality (Codex pass 1 Critical #3 反映)

Sprint 2.7c の implementation 開始前に下記の name / type collision を recon で確認 + design で resolution path 確定する必要あり:

1. **`seedExercises` 名 衝突**: `src/infra/database/connection.ts:42` で既に `seed/foods` から `seedExercises` を import しており、 これは legacy `exercises` table (workout 種目 master) 向け。 Phase 2.7c で search_index の 'exercise' source_type 向けに新 seed function を作るなら、 **`seedExerciseSearchIndex` 等の non-conflicting symbol name** を採用要。

2. **`seedExercisesV2` 並列存在**: `connection.ts:41` で `scripts/seed-exercises-v2/run` の `seedExercisesV2` も import 済。 これは exercises table の V2 系列で、 580 件 master と conceptually overlap する可能性。 Sprint 2.7c.1 で **legacy `seedExercises` + `seedExercisesV2` + Phase 2.7c 新規 search_index seed の 3 系列が並存する妥当性 vs 統合判断** が必要。

3. **`seedSearchIndex.ts` row type 拡張**: `src/infra/database/seed/searchIndex.ts:18-22` の `SearchIndexSeedRow` interface が `source_type: 'food' | 'restaurant_menu'` (literal union) で typed。 'exercise' を加えるには **union 拡張 + nutrition_json optional 化** (exercise rows は栄養 metric を持たない) が必要。

4. **`search_favorites` UI 消費の不在**: v38 migration の `search_favorites` table は production add.tsx で消費されておらず (Sprint 2.7.4 orphan cleanup で `useFavorite` + `FavoriteToggleButton` 削除済)、 ★ toggle UI は Phase 2.7c.5 で **新規 build 必要** (add.tsx の検索 sub-tab 内に exercise ★ コンポーネント新設)。

5. **production search 経路**: add.tsx の検索 sub-tab は `searchFoods` (foodRepository) + `searchDishes` (dishRepository) を使う。 exercise rows を hit させるには **新 repository (`searchExercises` or `searchIndexRepository` 復活)** が必要 — Sprint 2.7.4 で削除した `searchIndexRepository` を Phase 2.7c で **再 land する判断** が現実的 (schema 既 retain なので非破壊的、 ただし Sprint 2.7.4 削除と整合性矛盾を design で明示)。
