## Promotion sequence (Syuto-san)

Execute in order. Each step is reversible up to the App Store
submit.

1. **Push the v1.4 commits to origin**
   ```
   git push origin main
   ```
   At the time this checklist was written the local branch was
   ahead of origin by ~17 commits across Stages 1-5.4; re-run
   `git rev-list --count origin/main..HEAD` for the live number.
   Until the push lands, EAS will build off whatever origin has.

2. **Cut the release-candidate tag**
   ```
   git tag -a v1.4.0-rc1 -m "v1.4.0 release candidate 1"
   git push origin v1.4.0-rc1
   ```

3. **EAS production build**
   ```
   eas build --platform ios --profile production
   ```
   `appVersionSource: "remote"` + `autoIncrement: true` in
   `eas.json` handle the buildNumber automatically — no manual
   bump needed (Stage 5.1B cleaned the stale
   `ios.buildNumber: '1'` literal out of `app.config.ts` for
   this reason).

4. **TestFlight self-dogfood (24-48 h)**
   Install the resulting build via TestFlight. Re-run the v1.4
   QA scenarios end-to-end on the production build (not the dev
   build) to catch any release-config-only regressions.

5. **App Store Connect — v1.4.0 record**
   - Create a new version row.
   - Paste the user-facing portion of the v1.4.0 changelog into
     "What's New in This Version".
   - The architectural section of the changelog is for the repo,
     not for ASC.

6. **Submit for review**
   - Verify the export-compliance row reads "No" (the v1.4 ship
     does not add any new cryptographic functionality outside
     the standard HTTPS / Apple-frameworks set).
   - Submit.

## Rollback path

If a critical issue surfaces in TestFlight:

- The local `main` is the single source of truth for the
  implementation side. Revert the offending commit, re-tag,
  re-build. Do NOT amend or rewrite history that has already
  been pushed.
- If the App Store submission has already gone in, use the ASC
  "Reject this binary" flow before review picks it up.

## Post-ship hygiene

Once v1.4.0 is live:

  for any v1.4 carry-overs that shipped without a v1.5 deferral.
- Open the v1.5 recon thread off the freshly-tagged baseline.
